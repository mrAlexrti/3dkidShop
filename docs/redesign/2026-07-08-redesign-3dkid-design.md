# 3D Kid — дизайн-спецификация редизайна

Ветка: `redesign-3dkid-style`. Цель: современный, playful, pastel, kawaii, rounded 3D-облик
на основе `ескиз.jpg`, **без слома существующей бизнес-логики** (LiqPay, Nova Poshta,
Telegram, Resend, GitHub Actions, checkout flow, статусы заказов, API routes, env, URL).

## Согласованные решения

- **Картинки категорий:** текстовое поле URL + live-preview + fallback-заглушка. Без
  инфраструктуры загрузки файлов, без новых env-переменных, без новых зависимостей.
- **Хранение шрифта:** существующая модель `SiteContent`, ключ `site_font` (без изменения
  схемы под шрифты).
- **Дефолтные шрифты:** Fredoka (заголовки) + Nunito (текст). Курируемый список шрифтов с
  поддержкой кириллицы: Fredoka, Nunito, Comfortaa, Rubik, Manrope, Unbounded, Onest.
- **Персонажи Header:** кастомные kawaii-SVG (сердце с глазами, солнце, цветок-солнце,
  синий блоб), заменяют обычные эмодзи.

## Архитектура

### i18n (украинский по умолчанию + английский)
- Источник истины — **cookie** `locale` (`ua` | `en`), по умолчанию `ua`.
- `src/lib/i18n-server.ts`: `getLocale()` читает cookie через `next/headers`, `getT()`
  возвращает `translations[locale]` для server-компонентов/страниц/админки.
- Клиентские компоненты используют `useLangStore` (zustand). Стор инициализируется значением
  cookie, которое корневой layout прокидывает вниз, поэтому клиент и сервер согласованы.
- Переключение языка: `setLocale` (стор) + запись cookie + `router.refresh()`, чтобы
  server-компоненты перерисовались на новом языке.
- Middleware **не** трогается.
- Все захардкоженные пользовательские строки вынесены в `translations` (добавлена ветка
  `admin`).

### Шрифты
- Загружаются через `<link>` Google Fonts в корневом layout (грузится только выбранная тема).
  Активная тема читается из `SiteContent['site_font']`, применяется через CSS-переменные
  `--font-body`/`--font-display`, потребляемые Tailwind. Fallback = Fredoka + Nunito.

### Категории
- Prisma: добавлено **только** `Category.parentId` (self-relation) для подкатегорий. Ничего
  не переименовано и не удалено.
- Добавлен отсутствовавший server action `updateCategory`; `create/updateCategory` теперь
  принимают `parentId` + `imageUrl`.
- Канонический набор засеивается идемпотентно (upsert по slug):
  - `3d-toys` — 3D іграшки
  - `other-3d` — Інші 3D товари → подкатегории: `buttons` (Ґудзики),
    `crocs-jibbitz` (Джибітси для Crocs), `earphone-accessories` (Аксесуари для навушників)
  - `shoppers` — Шопери
  - `courses-3d` — Курси з 3D моделювання
- Существующие продовые категории **не** удаляются автоматически (деструктивно для живых
  данных); дедуп делается вручную через админку либо осознанным скриптом.

## Этапы (после каждого — стоп + пак файлов + lint/build)

1. **Фундамент** — cookie-бэкенд i18n + `getT`, словарь + ветка `admin`, подключение всех
   захардкоженных компонентов storefront, бренд STIKR→3D Kid; шрифты через `<link>` +
   динамический `site_font`; Prisma `Category.parentId` + `updateCategory` + идемпотентный
   seed. Требует `db push`.
2. **Header** — полный редизайн по эскизу (пухлый 3D-логотип, кастомные SVG-персонажи,
   карточки категорий/навигация из БД с подкатегориями, адаптив).
3. **Каталог** — новый rounded/playful UI поиска, стилизованная сортировка, фильтр категорий
   с подкатегориями.
4. **Админка + Главная + ревью** — обновлённый UI админки, preview/fallback картинок
   категорий, поле подкатегории, селектор шрифта, локализация админки; карточки категорий на
   главной; финальное самостоятельное код-ревью.

## Вне скоупа / нельзя ломать
LiqPay API, интеграция Nova Poshta, Telegram, Resend, workflow GitHub Actions, middleware
(без необходимости), env-переменные, существующие API routes, checkout flow, логика статусов
заказов, существующие Prisma-модели (только аддитивные изменения), существующие URL.
